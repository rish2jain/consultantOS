State Flow
==========

User Interaction → Form State Update → Validation
                                          ↓
                                     Valid? No → Show Errors
                                          ↓ Yes
                                     Clear Errors
                                          ↓
                                    API Request
                                          ↓
                            Success              Error
                               ↓                  ↓
                        Show Success Alert   Show Error Alert
                               ↓                  ↓
                         Call onSuccess      Call onError
                               ↓
                          Reset Form

Accessibility Features
======================

✓ Keyboard Navigation
  - Tab order follows visual hierarchy
  - Enter/Space for selections
  - Arrow keys for dropdown navigation
  - Escape to dismiss dropdowns

✓ Screen Reader Support
  - ARIA labels on all interactive elements
  - aria-describedby for error messages
  - aria-required for required fields
  - role="alert" for error announcements

✓ Visual Accessibility
  - 4.5:1 color contrast minimum
  - Focus indicators on all elements
  - Error states with icon + color + text
  - Required field indicators (*)

✓ Form Accessibility
  - Labels associated with inputs
  - Helper text for guidance
  - Character counters
  - Clear validation messages

Responsive Breakpoints
======================

Mobile (< 640px)
- Single column layout
- Full width inputs
- Stacked form actions
- 1 column depth selector

Tablet (640px - 1024px)
- Optimized spacing
- 2 column depth selector
- Side-by-side optional

Desktop (> 1024px)
- Max width: 4xl (896px)
- 3 column depth selector
- Side-by-side form actions
